# 移动端响应式优化方案

## 问题分析

从移动端截图发现的问题：
1. **转盘过大**：在小屏幕上占用过多空间
2. **布局不紧凑**：间距过大，内容显示不完整
3. **字体过大**：在小屏幕上显示效果不佳
4. **按钮尺寸**：不适配移动端手指操作

## 响应式设计方案

### 1. 屏幕尺寸断点
```typescript
const { width, height } = Dimensions.get('window');
const isSmallScreen = width < 400;      // 小屏手机
const isMediumScreen = width >= 400 && width < 600;  // 中等屏幕
// 大屏幕 width >= 600
```

### 2. 转盘组件响应式
```typescript
const getWheelSize = () => {
  if (screenWidth < 400) {
    return Math.min(screenWidth - 60, 240);  // 小屏：240px
  } else if (screenWidth < 600) {
    return Math.min(screenWidth - 80, 280);  // 中屏：280px
  } else {
    return Math.min(screenWidth - 100, 320); // 大屏：320px
  }
};
```

### 3. 动态样式系统
使用函数式样式生成，根据屏幕尺寸动态调整：

```typescript
const getStyles = () => StyleSheet.create({
  title: {
    fontSize: isSmallScreen 
      ? instagramTheme.typography.fontSize.title 
      : instagramTheme.typography.fontSize.hero,
  },
  // ...其他样式
});
```

```typescript
flexDirection: isSmallScreen ? 'column' : 'row',
```

### 3. 尺寸适配
```typescript
width: isSmallScreen ? 56 : 64,
height: isSmallScreen ? 56 : 64,
```

### 4. 字体缩放
```typescript
fontSize: isSmallScreen 
  ? instagramTheme.typography.fontSize.sm 
  : instagramTheme.typography.fontSize.md,
```

## 性能优化

### 1. 样式缓存
使用函数式样式生成，避免每次渲染重新计算：
```typescript
const getStyles = () => StyleSheet.create({ ... });
const styles = getStyles();
```

### 2. 条件渲染
只在必要时应用小屏幕样式：
```typescript
style={[styles.base, isSmallScreen && styles.small]}
```

### 3. 尺寸检测优化
在组件顶层进行一次尺寸检测，避免重复计算。
