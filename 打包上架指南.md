# 吃什么 App 打包上架指南（Expo + EAS）

本文档适用于当前项目（`expo@55`、`expo-router`），目标是完成：

- Android 打包（AAB/APK）
- iOS 打包（IPA）
- Google Play / App Store Connect 提交流程

---

## 1. 上架前准备

### 1.1 账号准备

- Expo 账号（用于 EAS Build）
- Google Play 开发者账号（一次性费用）
- Apple Developer 账号（年费）

### 1.2 本地环境准备

```bash
npm install
npm install -g eas-cli
eas login
```

### 1.3 代码与资源自检

- 应用图标、启动图已准备
- 文案无明显错别字
- 核心功能可用（新增餐点、转盘、列表）
- 隐私政策 URL 准备好（上架必需）

---

## 2. 必改配置（当前项目）

你当前 `app.json` 里缺少上架关键字段，建议补充如下：

```json
{
  "expo": {
    "name": "吃什么",
    "slug": "eatwhat",
    "version": "1.0.0",
    "ios": {
      "bundleIdentifier": "com.yourname.eatwhat",
      "buildNumber": "1"
    },
    "android": {
      "package": "com.yourname.eatwhat",
      "versionCode": 1
    }
  }
}
```

说明：

- `ios.bundleIdentifier`：iOS 唯一包名，创建后不要轻易改
- `android.package`：Android 包名，创建后不要轻易改
- `buildNumber` / `versionCode`：每次上新版本都要递增

---

## 3. 初始化 EAS（首次）

在项目根目录执行：

```bash
eas build:configure
```

该命令会生成 `eas.json`。建议使用如下配置（可直接参考）：

```json
{
  "cli": {
    "version": ">= 12.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal"
    },
    "production": {
      "autoIncrement": true
    }
  },
  "submit": {
    "production": {}
  }
}
```

---

## 4. Android 打包与上架

### 4.1 生成生产包（AAB）

```bash
eas build -p android --profile production
```

完成后会得到一个构建链接，下载 `.aab` 文件。

### 4.2 上传到 Google Play

1. 打开 Google Play Console
2. 创建应用（应用名称、默认语言、应用类型）
3. 进入「测试」或「正式版」轨道
4. 上传 `.aab`
5. 填写商店资料（描述、截图、隐私政策 URL、联系方式）
6. 提交审核

---

## 5. iOS 打包与上架

### 5.1 生成生产包（IPA）

```bash
eas build -p ios --profile production
```

首次会提示证书与描述文件，建议交给 EAS 托管（默认）。

### 5.2 提交到 App Store Connect

方式一（推荐，命令行）：

```bash
eas submit -p ios --profile production
```

方式二（网页）：

- 在 EAS 构建页面下载/提交到 TestFlight
- 进入 App Store Connect 完成版本信息并提交审核

---

## 6. 版本发布规范（建议）

### 6.1 版本号策略

- `version`：语义化版本（如 `1.0.1`）
- `android.versionCode`：整数递增（1,2,3...）
- `ios.buildNumber`：字符串数字递增（1,2,3...）

### 6.2 发布前检查清单

- [ ] 真机回归：新增、编辑、删除、转盘抽取
- [ ] 各机型 UI 适配（小屏/大屏、刘海屏）
- [ ] 崩溃和红屏清零
- [ ] 图标、启动图、应用名确认
- [ ] 隐私政策和用户协议可访问

---

## 7. 常见问题

### Q1：`versionCode` 已存在，无法上传？

`android.versionCode` 没递增。改成更大数字后重新构建。

### Q2：iOS 提审报缺少隐私信息？

到 App Store Connect 的 App Privacy 完整填写数据使用情况，并提供隐私政策链接。

### Q3：想先给测试同学安装？

使用内部分发：

```bash
eas build -p android --profile preview
eas build -p ios --profile preview
```

## 8. 推荐的发布命令顺序

```bash
# 1) 安装依赖
npm install

# 2) 本地检查
npm run start

# 3) Android 生产构建
eas build -p android --profile production

# 4) iOS 生产构建
eas build -p ios --profile production

# 5) iOS 提交（可选）
eas submit -p ios --profile production
```
