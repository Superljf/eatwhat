# 吃什么 EatWhat - 详细运行指南

本文档面向不熟悉 Expo 的开发者，提供从零到运行项目的完整步骤。

---

## 一、环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| **Node.js** | 18.x 或更高 | 运行 JavaScript 的运行时 |
| **npm** | 9.x 或更高 | 随 Node.js 安装，用于安装依赖 |
| **Git** | 任意 | 用于克隆/管理代码（可选） |

检查是否已安装：

```bash
node -v    # 应显示 v18.x 或更高
npm -v     # 应显示 9.x 或更高
```

若未安装，请前往 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本。

---

### 1.2 按运行方式选择额外环境

| 运行方式 | 额外需要 | 适用场景 |
|----------|----------|----------|
| **Web 浏览器** | 无 | 最快体验，无需额外配置 |
| **Expo Go 真机** | 手机 + Expo Go App | 在真实手机上体验 |
| **Android 模拟器** | Android Studio | 在电脑上模拟 Android |
| **iOS 模拟器** | macOS + Xcode | 仅 Mac 用户 |

---

## 二、安装依赖

在项目根目录 `eatwhat` 下打开终端，执行：

```bash
cd d:\2026Code\eatwhat
npm install
```

首次安装可能需要 1–3 分钟，完成后会生成 `node_modules` 文件夹。

如遇网络问题，可尝试使用国内镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

## 三、启动开发服务器

```bash
npm start
```

或直接：

```bash
npx expo start
```


## 四、运行方式详解

### 4.1 方式一：Web 浏览器（推荐入门）

**步骤：**

1. 在终端中按 `w`，或执行 `npm run web`
2. 浏览器会自动打开 `http://localhost:8081`（端口可能不同）
3. 在浏览器中即可使用 APP

**优点：** 无需手机、模拟器，启动快  
**注意：** 部分原生能力（如震动）在 Web 上可能无效

---

### 4.2 方式二：Expo Go 真机预览

**步骤：**

1. 在手机应用商店搜索并安装 **Expo Go**
   - [Android 应用商店](https://play.google.com/store/apps/details?id=host.exp.exponent)
   - [iOS App Store](https://apps.apple.com/app/expo-go/id982107779)

2. 确保手机和电脑在同一 Wi‑Fi 下

3. 在终端显示的二维码处：
   - **Android**：用 Expo Go 扫描二维码
   - **iOS**：用系统相机扫描，再选择「在 Expo Go 中打开」

4. 等待加载完成，即可在手机上使用

**优点：** 真实设备体验，支持震动等原生功能  
**注意：** 需同一局域网，部分公司网络可能限制连接

---

### 4.3 方式三：Android 模拟器

**前置准备：**

1. 安装 [Android Studio](https://developer.android.com/studio)
2. 打开 Android Studio → More Actions → Virtual Device Manager
3. 创建并启动一个虚拟设备（如 Pixel 6）

**步骤：**

1. 启动模拟器后，在终端按 `a`
2. 或执行：`npm run android`
3. 应用会自动安装到模拟器并打开

---

### 4.4 方式四：iOS 模拟器（仅 macOS）

**前置准备：**

1. 安装 Xcode（从 App Store 或开发者官网）
2. 打开 Xcode，执行一次 `xcode-select --install` 安装命令行工具

**步骤：**

1. 在终端按 `i`
2. 或执行：`npm run ios`
3. iOS 模拟器会自动启动并打开应用

## 六、常见问题

### 6.1 端口被占用

**现象：** 提示 `Port 8081 is being used`

**解决：** 指定其他端口：

```bash
npx expo start --port 8082
```

### 6.2 依赖安装失败

**现象：** `npm install` 报错或卡住

**解决：**

```bash
# 清除缓存后重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 6.3 真机无法连接

**现象：** 扫码后无法加载

**解决：**

```bash
# 使用 tunnel 模式（需安装 @expo/ngrok）
npx expo start --tunnel
```

### 6.4 Web 报错 `babel-preset-expo`

**现象：** 打包时报错找不到 babel-preset-expo

**解决：**

```bash
npm install babel-preset-expo --save-dev
```

### 6.5 修改代码后不自动刷新

在终端按 `r` 手动刷新，或检查是否开启了 Fast Refresh（Expo 默认开启）。
